# 04 视觉系统与组件规范

## 1. 风格

“教育创作工作室”：专业、安静、内容优先。不得使用传统统计后台、暗色科技驾驶舱、过度卡通化或重度玻璃拟态。

## 2. 色彩

| 页面背景 | `#F6F7FA` |
| 一级表面 | `#FFFFFF` |

## 3. 字体

```text
Inter, PingFang SC, Microsoft YaHei, Noto Sans SC, sans-serif
```

页面标题24/600；模块标题18/600；卡片标题16/600；正文14/400；辅助12/400。教案正文16px、行高1.75。提示词编辑器使用等宽字体。

## 4. 间距、圆角和阴影

- 4px基础网格。
- 页面间距24px。
- 面板内边距16或20px。
- 按钮和输入框圆角8px。
- 卡片10px，主面板12px，弹窗14px，大型预览16px。
- 普通卡片默认无阴影。
- 只在悬浮层、拖拽和弹窗使用阴影。

## 5. 组件层级

### 基础组件

Button、IconButton、Input、Textarea、Select、Checkbox、Radio、Switch、Tabs、Tooltip、Popover、Dialog、Drawer、Dropdown、Progress、Skeleton和Toast。

### 复合组件

PageHeader、ProjectSwitcher、LessonSwitcher、WorkflowRail、NodeHeader、PromptEditor、ModelSelector、CostEstimate、ValidationPanel、VersionTimeline、AssetPicker、GenerationTaskDock、CandidateGallery、ApprovalBar、EmptyState和ErrorRecoveryPanel。

### 业务组件

TextbookEvidenceViewer、LessonDivisionEditor、LessonPlanEditor、IntroOptionBoard、PptOutlineEditor、PptSlideEditor、StoryboardTimeline、ShotEditor、ClipReviewPlayer、DeliveryChecklist、ProviderHealthCard和RoutePolicyEditor。

业务页面不得直接导入Radix原始组件；必须使用项目封装层。

## 6. 操作层级

- Primary：生成、批准、确认创建。
- Secondary：保存、预览、重新校验。
- Outline：设置、比较、导出。
- Ghost：低优先辅助操作。
- Destructive：取消任务、撤销发布和删除允许删除的草稿。

每个页面同一时间只能有一个最高优先级操作。

## 7. 动效

- 按钮反馈120ms。
- 标签切换160ms。
- 面板展开180ms。
- 抽屉弹窗220ms。
- 页面局部切换240ms。
- 缓动统一为`cubic-bezier(0.2, 0.8, 0.2, 1)`。
- 支持`prefers-reduced-motion`。

禁止大面积弹跳、无限背景动画、发光文字、频繁流动渐变和所有卡片悬浮上移。

## 8. Figma要求

- 使用变量定义颜色、字号、间距、圆角、阴影和动效时长。
- 组件必须使用Auto Layout。
- 必须提供组件变体、交互状态和响应式示例。
- 页面不得大量使用脱离组件的局部样式。
- Figma命名应与代码组件命名对应。
- 交付源文件必须可编辑，不接受只有截图或只读链接。

## 9. 禁止设计

- 大面积无业务意义统计卡。
- 低对比度浅灰文字。
- 弹窗套弹窗。
- 图标按钮无Tooltip和文字解释。
- 把技术字段直接展示给普通教师。
- 生成期间遮住整个页面。
- 为追求简洁隐藏版本、费用、失败和阻断原因。
