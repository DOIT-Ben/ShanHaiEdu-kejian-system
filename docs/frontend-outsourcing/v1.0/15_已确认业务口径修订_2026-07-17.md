# 15 已确认业务口径修订（2026-07-17）

状态：强制执行  
适用对象：产品、UI/UX、前端开发、Mock、联调、测试和验收  
优先级：高于本外包包内00—14号文档和`contracts/`需求草案

## 1. 为什么有这份修订

外包包V1.0形成后，项目、创作台、动态内容结构、PPT内部链路、视频顺序、状态机和API响应合同已经完成后续确认。旧文档保留版本号用于交付追溯，但实现必须应用本修订和正式运行时设计：

`docs/superpowers/specs/2026-07-17-shanhai-courseware-platform-runtime-design.md`

## 2. 必须修改的产品口径

| 旧口径 | 当前口径 |
|---|---|
| 一个教材父项目创建多个课时子项目 | 一个Project对应一个小知识点教材，内部包含多个LessonUnit；课时不是独立Project |
| 教案固定十二部分 | 十二部分是内置默认内容结构；页面必须由Content Definition Schema动态渲染 |
| 创作台挂载或附着在项目节点 | 创作台是独立通用能力；项目以CreationPackage导入，结果以SaveToProjectOperation保存回项目 |
| 创作台候选自动进入项目资产 | 候选默认只属于创作任务；用户选择并保存后才进入指定项目资产包 |
| 视频只读取导入方案和锚点 | 锚点选择阶段隔离课程内容；锚点批准后，故事阶段读取受控CourseKnowledgeBrief |
| 视频先定视觉再粗分镜 | 完整母版故事→粗分镜→视觉母图/视觉契约→图片资产→细分镜→shot候选→合格clip→合成 |
| 一个模型返回视频即是clip | 返回的是候选；通过技术校验、被选中并保存到项目shot资产位后才产生正式clip_id |
| PPT可按整页图片直接交付 | 整页图片用于预览；最终PPTX默认保持文字、公式、数据、题目和关键标签可编辑 |
| 单一节点状态覆盖全部过程 | NodeRun、GenerationJob、候选审核、保存和Approval分别显示状态 |
| 成功响应包含`ok` | 当前标准为`data/meta/request_id`；错误为`error/request_id` |

## 3. 前端必须具备的动态结构能力

教案、课时划分、PPT大纲、PPT逐页规格、故事和分镜页面不能以固定字段数组实现。前端接收：

- `content_definition_version_id`
- `schema`
- `ui_schema`
- `data`
- 字段权限和必填等级
- 字段级校验结果

通用渲染器必须支持text、rich_text、number、list、table、group、repeatable、reference、asset、math、rubric和timeline。未知字段类型显示不兼容错误，不得静默忽略。

字段级别至少显示：硬必填、软必填、可选、禁用；可编辑、可删除、可隐藏、可排序和可局部AI返修权限。硬必填删除要阻止或要求符合后端规则，软必填删除要明确风险。

## 4. 创作台页面关系

全局导航新增或保留平台级入口：

- 图片创作台
- 视频创作台
- PPT创作台

进入创作台有两种方式：

1. 独立进入，用户手工创建批次。
2. 从项目选择待创作项，一键生成CreationPackage并导入批次。

导入后页面显示来源项目和默认目标资产位，但不表现为“项目已挂载”。用户可以修改任务，并在生成后选择保存到来源项目、其他有权限项目或只下载。

批量生图、生视频和PPT页面采用“一批次＋N张任务卡＋统一任务坞”，不得打开N个独立工作台。每张卡支持候选比较、单项重试、选择和保存状态。

## 5. 项目工作台关系

项目总览显示教材、课时和各课时分支。每个LessonUnit拥有教案、可选PPT和可选视频；分支可以disabled、skipped或重新开启。

项目资产库展示的是已保存项目资产和中间正式版本，不展示尚未保存的全部创作台候选。项目节点提供“准备创作包”和“一键导入创作台”，不复制创作台生成界面。

## 6. Prompt面板

所有模型节点继续使用：输入与选择、系统组装结果、可编辑业务Prompt、只读平台约束摘要、输出Schema摘要、模型参数、成本预估、输出面板、审核返修和版本记录。

用户可以修改业务Prompt，但不能删除安全约束、ContextBinding白名单、输出Schema和资产白名单。提交后展示不可变PromptSnapshot编号。

## 7. PPT页面修订

PPT子流程固定为：大纲、逐页规格、封面门禁、视觉契约、正文页面资产、页面预览、PPTX装配、最终审核。

封面通过前，正文批量生成按钮不可用。正文画布默认纯白。页面编辑区需要区分：可编辑文本/数学对象层、图片资产层和整页预览。单页修改只标记该页预览和当前PPTX版本过期。

## 8. 视频页面修订

视频子流程固定为：三类九套、锚点与主题、母版故事、粗分镜、视觉母图、图片资产、细分镜、镜头任务、clip选择、声音字幕、成片。

粗分镜是故事节拍时间线，不显示为最终模型Prompt。视觉母图批准后显示VideoStyleContract。细分镜卡显示稳定`shot_id`、已批准图片、动作、运镜、时长、声音意图、最终视频Prompt和候选结果。候选保存成功后才显示正式`clip_id`。

## 9. 状态和实时任务

NodeRun状态采用：`disabled`、`not_ready`、`ready`、`draft`、`queued`、`running`、`review_required`、`approved`、`partially_completed`、`failed`、`paused`、`cancel_requested`、`cancelled`、`stale`、`skipped`。

任务卡单独显示技术状态、审核状态和保存状态。SSE端点优先使用`/api/v2/events/stream`、项目流和任务流；支持Last-Event-ID。SSE只触发Query刷新，不能直接覆盖完整业务对象。

## 10. API草案状态

`contracts/frontend-api-requirements.openapi.yaml`仍是外包开发Mock需求草案，不是正式后端合同。前端必须保留API适配层和生成客户端。联调时以后端提交的正式OpenAPI 3.1替换草案，禁止把草案DTO散落复制到组件。

## 11. 验收增量

除原验收项外，新增：

- 切换两套不同教案Schema无需修改页面代码。
- 项目创作包可导入通用创作台，结果可保存回来源或其他有权限项目。
- 未保存候选不污染项目资产库。
- PPT最终交付中关键课堂内容可编辑。
- 视频顺序、shot/clip正式化和局部返修符合当前口径。
- 页面能同时表达节点、任务、审核和保存四类状态。
- Prompt可编辑层与只读约束层明确分离。

承接方如已经按旧口径实现，必须将受影响项列入变更清单；本修订属于业务基线纠偏，不得通过继续固化旧数据结构规避。
