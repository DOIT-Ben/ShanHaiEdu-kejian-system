# 11 外包交付物与验收标准

> 当前业务修订见`15_已确认业务口径修订_2026-07-17.md`；增量验收项同样属于交付范围。

## 1. 设计交付

- 完整Figma源文件，归甲方账号所有。
- 设计变量：颜色、字号、间距、圆角、阴影和动效。
- 基础、复合和业务组件库。
- 1440、1280和1024关键页面。
- 手机查看与审批页面。
- 空、加载、成功、警告、错误、阻断、失效和部分成功状态。
- 可点击交互原型。
- 组件、页面和交互说明。

验收不接受只有截图、PDF或只读演示链接。

## 2. 代码交付

- 完整Git仓库和提交历史。
- React 19＋TypeScript＋Vite生产源码。
- pnpm锁文件。
- 全部环境变量模板。
- OpenAPI生成客户端。
- MSW Mock和fixtures。
- Storybook。
- 单元、组件、契约和E2E测试。
- CI配置。
- 开发、构建、测试和部署README。
- 第三方依赖及许可证清单。

## 3. 功能交付

### 教师端

- 登录和会话恢复。
- 首页和项目列表。
- 新建教材项目。
- 教材证据。
- 课时划分。
- Schema驱动教案编辑器，内置默认十二部分并能无代码切换第二套测试结构。
- 三类九套导入设计。
- 平台级图片、视频和PPT创作台，以及项目创作包导入和结果保存回项目。
- PPT工作区。
- 视频完整工作区。
- 提示词、模型、费用和版本。
- 项目资产库。
- 任务中心和SSE。
- 交付中心。

### 管理端

- 模板中心。
- 模型网关总览。
- Provider和模型目录。
- 路由策略。
- 预算额度。
- 调用记录。
- 工作流版本。
- 用户机构和审计页面。

## 4. 技术验收

- 全新环境可按README安装。
- `pnpm install --frozen-lockfile`成功。
- Mock模式不依赖后端即可运行。
- 真实API模式不包含Mock响应。
- lint、typecheck、test、build和E2E全部通过。
- 页面刷新、深链接和浏览器返回正常。
- 无未处理Promise错误和控制台错误。
- 无`any`滥用、硬编码密钥、生产Mock和业务页面直接fetch。
- SSE断线重连和轮询降级有效。
- 409冲突、预算不足和Provider失败可操作。
- 生产构建不包含Source Map公开敏感源码，是否上传私有监控由甲方配置。

## 5. 视觉验收

- 与批准Figma在目标浏览器和尺寸下保持一致。
- 页面没有横向溢出、文字截断和不可见焦点。
- 中文长文本、长项目名和长错误文案不破坏布局。
- PPT正文预览为纯白。
- 状态不只依赖颜色。
- 不出现未批准的玻璃拟态、重度渐变或暗色全局主题。

## 6. 质量验收

- 满足`10_测试性能安全与可访问性.md`。
- 核心E2E路径全部通过。
- P0、P1缺陷为零。
- P2缺陷有书面处理计划且不阻断核心流程。
- 关键页面通过视觉回归。
- 依赖漏洞扫描无已知严重和高危未处理项。

## 7. 交接验收

- 源码位于甲方控制的仓库。
- Figma位于甲方控制的空间。
- CI、部署和监控账号归甲方。
- 外包账号和临时权限已清理。
- 至少两次技术交接会议并提供录像或纪要。
- 甲方工程师独立完成一次安装、Mock运行、真实API联调和生产构建。
- 承接方提供不少于90天缺陷保修期。

## 8. 知识产权与安全建议

- 甲方支付对应里程碑后取得该里程碑全部源代码、设计和文档权利。
- 承接方不得将项目代码、教学数据、提示词和接口资料用于其他客户或模型训练。
- 未经书面同意不得转包。
- 承接方只能使用脱敏Mock数据。
- 不向承接方提供模型生产密钥和真实教师密码。
- 发现安全事件应在约定时限内报告并保留证据。
