## 3. 前后端边界

前端只访问版本化业务API：

```text
/api/v2/*
```

前端不得：

- 直接调用任何文本、图片、视频或TTS模型API。
- 保存完整API Key。
- 自行判断正式节点状态。
- 自行计算正式费用。
- 通过前端逻辑绕过预算、权限、审核或质量门禁。
- 将模型输出直接当作可信HTML执行。

## 4. 固定产品决策

- 第一版闭环小学数学，架构预留其他学科。
- 一个项目对应一个小知识点教材，项目内部包含多个稳定课时；课时不是独立项目。
- 每个课时独立拥有教案以及可选PPT、视频和交付分支。
- 教案由可导入的内容结构Schema驱动，系统内置默认模板为十二部分。
- 三类九套是独立产物，不放进教案正文。
- 独立创意生成不读取教材、知识点、教案和PPT。
- 课程侧只能通过科普、应用、故事三类锚点合同连接独立创意。
- 锚点批准后，故事节点可以读取受控课程知识摘要完成适配。
- 视频顺序固定为母版故事、粗分镜、视觉母图与视觉契约、图片资产、细分镜和镜头提示词、候选片段、合格clip、声音字幕和剪辑成片。
- 一个镜头允许多次生成，但只有一个当前批准片段。
- 图片、视频和PPT创作台是平台级能力，不与项目长期挂载；项目用创作包导入，结果经保存操作回填。
- 模型返回的视频是候选，技术合格、被选中并保存后才产生正式clip_id。
- 提示词在前端可见、可编辑、可保存版本。
- 下游引用具体资产版本；上游变化只标记失效，不删除历史产物。
- 专业手动和半自动是主要模式；全自动只生成待审核草稿。

## 5. 固定视觉决策

- 视觉方向为“教育创作工作室”。
- 大面积暖白和纯白，山海蓝为主色。
- 不采用传统统计后台、玻璃拟态、重度渐变和暗色驾驶舱。
- 第一版不做全局暗色模式。
- PPT正文预览必须保持纯白背景。
- PPT整页图片仅作为预览；最终PPTX默认保持文字、公式、数据、题目和关键标签可编辑。
- 重点适配1440×900和1280px宽桌面。
- 手机只提供查看、审批和预览，不承担完整制作。

## 6. 承接方可提出建议但不得自行改变

- 微文案润色。
- 图标细节。
- 空状态插画。
- 不改变结构的局部间距优化。
- 不改变业务合同的动效细节。

涉及页面结构、路由、业务状态、API、组件行为、颜色主体系、模型费用提示或审批机制的变化，必须走书面变更流程。
