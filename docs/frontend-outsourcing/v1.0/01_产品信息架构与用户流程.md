# 01 产品信息架构与用户流程

> 当前业务修订见`15_已确认业务口径修订_2026-07-17.md`。

## 1. 用户角色

### 教师

- 创建教材项目。
- 上传教材并审核教材证据。
- 审核课时划分。
- 生成、编辑和批准教案。
- 生成并选择三类九套导入设计。
- 制作PPT和视频。
- 管理项目资产、任务和交付。

### 模板管理员

- 查看、导入、试运行、发布、绑定和回滚节点模板。
- 查看模板校验和使用情况。

### 系统管理员

- 管理模型网关、Provider、模型、路由、预算和健康状态。
- 管理用户、机构、工作流版本和审计日志。

## 4. 核心教师流程

```text
登录
→ 新建教材项目
→ 上传教材
→ 审核教材证据
→ 审核课时划分
→ 创建多个课时
→ 进入单课时工作台
→ 按当前内容结构生成并批准教案（默认十二部分）
→ 生成三类九套并批准一套
→ 可选生成PPT
→ 可选生成视频
→ 检查交付清单
→ 下载交付包
```

项目需要批量生图、生视频或生成PPT时，流程为：

```text
项目选择待生成项
→ 创建不可变CreationPackage
→ 一键导入平台级创作台
→ 编辑并批量生成
→ 选择候选结果
→ 保存回来源或其他有权限项目
```

## 5. 统一模型生成流程

```text
填写或选择输入
→ 系统准备提示词草稿
→ 教师查看或修改提示词
→ 选择授权范围内的模型策略
→ 查看预计费用
→ 点击生成
→ 任务排队或运行
→ 展示候选结果
→ 展示机器校验
→ 教师批准、返修或重新生成
```

## 6. 用户体验原则

- 首页优先“继续工作”，不优先统计数据。
- 工作流步骤使用教师语言，不展示内部节点ID。
- 当前输出始终位于中心画布。
- 输入、提示词、参数、校验、版本和资产位于右侧检查器。
- 任何付费生成前展示模型、候选数量和预计费用。
- 长任务离开页面继续运行。
- 历史版本可查看、比较和恢复，批准版本不可原地覆盖。
- 错误信息必须包含下一步解决方法。
