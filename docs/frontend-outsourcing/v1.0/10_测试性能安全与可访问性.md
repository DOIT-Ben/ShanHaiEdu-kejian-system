# 10 测试、性能、安全与可访问性

## 1. 测试分层

### 单元测试

- 状态映射。
- 费用显示。
- 错误解析。
- 路由权限。
- 版本冲突决策。
- 文件大小和类型校验。
- SSE事件解析与重连退避。

### 组件测试

- 基础组件全部状态。
- WorkflowRail。
- PromptEditor。
- VersionTimeline。
- AssetPicker。
- GenerationTaskDock。
- IntroOptionBoard。
- StoryboardTimeline。
- ShotEditor。
- Provider和RoutePolicy表单。

### 契约测试

- OpenAPI生成成功。
- Mock响应符合生成类型和Schema。
- 错误Envelope可解析。
- SSE事件符合Schema。

### 端到端测试

至少覆盖：

1. 登录并创建教材项目。
2. 上传教材并进入解析状态。
3. 批准课时划分并进入课时。
4. 修改提示词、生成和批准教案。
5. 选择三类九套中的一个方案。
6. PPT关键样张审核。
7. 视频镜头候选批准和失败重试。
8. 交付包生成和下载。
9. 管理员配置Provider、模型和路由策略。
10. 会话过期、预算不足、版本冲突和SSE断线恢复。

## 2. CI门禁

每个PR必须通过：

```text
pnpm lint
pnpm typecheck
pnpm test --run
pnpm build
pnpm test:e2e --project=chromium
```

主分支和发布候选还需通过完整浏览器矩阵和视觉回归。

## 3. 覆盖率

- shared/lib、API错误、状态映射和路由守卫：行覆盖率90%以上。
- features和widgets：行覆盖率80%以上。
- 关键生成、批准、预算和版本冲突路径必须有行为测试，不以总覆盖率代替。

## 4. 性能指标

在生产构建、常规办公网络和1440px桌面下：

- 首次进入登录后首页LCP不高于2.5秒。
- 交互响应INP不高于200ms。
- CLS不高于0.1。
- 路由切换骨架反馈在100ms内出现。
- 10MB图片列表采用缩略图和虚拟化，不加载原图。
- 100个镜头卡或500个资产条目滚动保持可用。
- 大文件哈希和处理不阻塞主线程。
- 初始页面不得打包所有PPT和视频编辑器，按路由和节点懒加载。

## 5. 浏览器

- Chrome当前稳定版及前两个主要版本。
- Edge当前稳定版及前两个主要版本。
- Safari当前稳定版及前一个主要版本。
- 平板Safari用于查看、审核和轻量编辑。

不要求支持IE和过旧浏览器。

## 6. 安全

- 不在localStorage保存会话Token。
- 不提交`.env`和密钥。
- 不执行模型输出HTML或脚本。
- 外链使用安全属性。
- 文件名和Markdown严格转义。
- 权限判断以前后端为准，前端权限只用于隐藏和引导。
- 管理端高风险操作需要二次确认和操作理由。
- API Key输入提交后从组件状态清除。
- 错误监控不上传教材全文、提示词全文和敏感个人信息。

## 7. 可访问性

- 目标WCAG 2.1 AA。
- 文本和背景对比度满足AA。
- 全部核心流程支持键盘。
- 焦点顺序与视觉顺序一致。
- 弹窗和抽屉具有焦点锁定和关闭恢复。
- 图标按钮有可访问名称。
- 表单错误与字段关联。
- 视频有字幕和键盘控制。
- 支持`prefers-reduced-motion`。

## 8. 视觉回归

必须覆盖：

- 1440、1280、1024三种宽度。
- 工作台各检查器和任务坞状态。
- 基础组件全部变体。
- 两套动态教案Schema、三类九套、平台级创作台、PPT、视频和模型网关核心页面。
- 空、加载、错误、阻断、失效和部分成功状态。
