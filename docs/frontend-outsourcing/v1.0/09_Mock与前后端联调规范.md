# 09 Mock与前后端联调规范

## 1. 目标

承接方在没有后端源码和生产凭据的情况下完成全部前端开发。Mock不得另造一套DTO，必须基于同一OpenAPI生成类型。

## 2. 运行模式

```text
pnpm dev:mock       全部接口由MSW提供
pnpm dev            连接开发后端
pnpm test           测试环境使用确定性Mock
pnpm build          生产构建，Mock代码不启用
```

## 3. Mock目录

```text
src/mocks/
├─ browser.ts
├─ handlers/
│  ├─ auth.ts
│  ├─ projects.ts
│  ├─ workflows.ts
│  ├─ artifacts.ts
│  ├─ tasks.ts
│  ├─ templates.ts
│  └─ model-gateway.ts
├─ fixtures/
├─ scenarios/
└─ event-stream/
```

## 4. 必备场景

- 登录成功、失败、锁定和会话过期。
- 无项目、单项目和多项目。
- 教材上传、解析、部分识别和解析失败。
- 正常课时划分、教师修改和版本冲突。
- 十二部分教案生成、返修和批准。
- 三类九套完整、单套失败和锚点校验失败。
- PPT页面、图片候选、关键样张和导出。
- 视频各节点、单镜头失败、片段重试和最终剪辑。
- 任务排队、运行、完成、失败、取消和重连恢复。
- 预算不足、限流、Provider降级和付费回退确认。
- 401、403、404、409、413、422、429和5xx。
- SSE断线和轮询降级。
- 部分候选成功。
- 上游批准版本变化导致下游失效。

具体场景ID见`contracts/mock-scenarios.json`。

## 5. 联调流程

1. 后端冻结一个OpenAPI版本。
2. 前端更新`contracts/openapi.json`并重新生成类型。
3. CI检查生成代码无未提交变化。
4. 前端使用开发环境账号联调。
5. 发现契约差异先更新OpenAPI，不在前端增加临时兼容字段。
6. 前后端共同完成关键路径联调清单。
7. 预发布环境使用真实对象存储和至少一套真实文本、图片、视频Provider冒烟。

## 6. 契约变更

- 删除字段、修改含义和修改类型属于破坏性变更。
- 新增可选字段属于兼容变更。
- 新增必填字段需要新API或合同版本。
- 前端不接受没有OpenAPI和示例的接口变更。
- 紧急兼容必须有移除日期和责任人。

## 7. 前端不得模拟为成功的行为

- 正式费用结算。
- Provider真实可用性。
- 文件病毒扫描。
- 教材解析质量。
- PPTX和视频技术门禁。
- 最终批准状态。
- 正式下载授权。

Mock页面必须具有明显的开发环境标识，生产构建不得启用。
