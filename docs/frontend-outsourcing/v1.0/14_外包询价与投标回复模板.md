# 14 外包询价与投标回复模板

## 1. 甲方询价说明

项目名称：山海教育课件工作台 V2 独立前端。  
交付形式：Figma设计＋React生产源码＋Mock＋测试＋文档＋交接。  
后端形式：甲方FastAPI，通过版本化OpenAPI和`/api/v2`联调。  
报价基线：本资料包全部文件。

承接方不得以“接口未完成”为由省略Mock模式、错误状态、测试和可访问性；甲方应按里程碑提供冻结的接口合同和联调环境。

## 2. 承接方必须回复

### 公司和团队

- 公司主体和成立时间。
- 最近三年类似AI工作台、编辑器或复杂业务前端案例。
- 本项目人员姓名、角色、投入比例和所在地。
- 是否使用分包或自由职业者。
- 前端负责人和设计负责人是否全程参与。

建议最低角色配置：

- 1名前端技术负责人。
- 1名产品/UI设计师。
- 2名前端工程师。
- 1名测试工程师，可阶段投入。
- 1名项目经理，可由负责人兼任。

### 技术方案

- 如何落实React＋Vite前后端分离。
- 如何组织Feature、Query、API生成代码和Storybook。
- 如何处理长文档、长列表、图片和视频性能。
- 如何处理SSE断线、轮询降级和后台任务。
- 如何保证Mock与OpenAPI一致。
- 如何执行视觉回归、可访问性和浏览器测试。
- 如何防止模型密钥和生产数据进入前端。

### 计划

- 里程碑时间。
- 每个里程碑人员投入。
- 甲方需要提供的输入和最晚日期。
- 关键风险和缓解方案。
- 90天保修期内响应方式。

## 5. 技术面试建议

要求候选前端负责人现场说明：

1. 为什么服务端状态不应复制到Zustand。
2. SSE和TanStack Query如何配合。
3. 如何让OpenAPI、Mock和测试共用类型。
4. 如何实现一个shot多个clip候选但只有一个批准版本。
5. 如何处理自动保存的409版本冲突。
6. 如何避免大图片和视频拖垮工作台。
7. 如何防止Provider密钥出现在浏览器。

无法清晰回答上述问题的团队不适合承担本项目核心前端。

## 6. 试做建议

正式签约前可安排付费小试做，范围限定为：

- 工作台四区布局。
- WorkflowRail。
- PromptEditor。
- 三类九套卡片中的一列。
- 一个SSE任务状态变化。
- Storybook和基础测试。

试做代码必须进入甲方临时仓库；不得要求提供后端源码和生产密钥。试做只评估能力，不替代正式里程碑验收。
