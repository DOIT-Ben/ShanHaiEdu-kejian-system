# 山海教育课件工作台 V2 前端外包交付包

> 强制修订：开工或继续开发前，必须先阅读`15_已确认业务口径修订_2026-07-17.md`。该修订高于本包00—14号文档和contracts需求草案，解决动态教案结构、项目与创作台、PPT、视频顺序、状态与API口径。

版本：1.0  
日期：2026-07-16  
项目形态：生产级独立前端  
目标技术栈：React 19＋TypeScript＋Vite

## 1. 用途

本资料包用于外包询价、产品设计、前端开发、前后端联调、测试验收和最终源码交接。承接方应以本包为范围基线，不得以旧版山海教育前端页面作为布局、视觉或代码结构参考。

## 2. 项目目标

建设一个面向小学数学课件生产的 AI 创作工作台，将教材解析、课时划分、Schema驱动教案（内置默认十二部分）、三类九套锚点、PPT、视频、通用创作台、项目资产、模型任务和最终交付集中在一个前端系统中。

前端必须做到：

- 教师不需要在多个模型网站之间切换。
- 所有模型输入、提示词、参数、输出、版本和费用都可见。
- 上游批准产物能够作为下游节点的版本化输入资产。
- 长耗时图片和视频任务可以离开页面继续运行。
- 普通教师使用业务语言，管理员可以管理模板、模型网关、预算和工作流版本。

## 3. 固定技术架构

- React 19。
- TypeScript严格模式。
- Vite。
- React Router。
- Tailwind CSS。
- shadcn/ui＋Radix UI。
- TanStack Query。
- Zustand，仅用于界面状态。
- React Hook Form＋Zod。
- Framer Motion。
- openapi-typescript＋openapi-fetch。
- MSW。
- Vitest＋Testing Library＋Playwright。
- Storybook。
- pnpm及锁文件。

不得引入Next.js、服务端BFF、第二套业务数据库或浏览器直连模型Provider。
