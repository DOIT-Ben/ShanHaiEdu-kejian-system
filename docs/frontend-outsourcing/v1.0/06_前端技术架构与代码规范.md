# 06 前端技术架构与代码规范

## 1. 技术栈

- React 19。
- TypeScript严格模式，`noImplicitAny: true`。
- Vite。
- React Router。
- Tailwind CSS。
- shadcn/ui＋Radix UI。
- TanStack Query。
- Zustand。
- React Hook Form＋Zod。
- Framer Motion。
- openapi-typescript＋openapi-fetch。
- MSW。
- Vitest＋Testing Library＋Playwright。
- Storybook。
- pnpm。

除安全修复外，承接方新增核心依赖前必须提交依赖用途、许可证、包体影响和替代方案。

## 2. 推荐目录

```text
src/
├─ app/                  启动、Provider、路由、错误边界
├─ pages/                路由页，只做页面编排
├─ layouts/              教师、工作台、管理端布局
├─ features/             可独立测试的业务动作
├─ entities/             项目、课时、节点、资产等实体展示
├─ widgets/              页面级组合模块
├─ shared/
│  ├─ ui/                基础组件
│  ├─ api/               生成客户端、错误、SSE
│  ├─ config/            运行配置和特性开关
│  ├─ hooks/
│  ├─ lib/
│  └─ styles/
├─ mocks/                MSW handlers与fixtures
└─ test/                 测试工具
```

模块使用公开`index.ts`出口，禁止跨模块深层导入和循环依赖。

## 3. 状态边界

### TanStack Query

保存所有服务端状态：会话、项目、课时、工作流、节点、提示词、产物、资产、任务、模型运行、模板和网关配置。

### Zustand

只保存界面状态：导航折叠、检查器宽度和标签、任务坞展开、临时比较、预览模式和本地布局偏好。

禁止把服务端项目和节点数据复制进Zustand。

### URL

保存项目ID、课时ID、节点Key、子页面和可分享筛选条件。

### 表单

使用React Hook Form＋Zod。长文档编辑器使用本地草稿和后端自动保存，但正式状态仍来自服务端。

## 4. API访问

- 页面不得直接调用`fetch`。
- API DTO由OpenAPI生成。
- 统一API客户端处理Cookie、CSRF、幂等键、超时、Trace ID和错误。
- 统一Query Key工厂。
- Mutation成功后精确失效相关查询，禁止无差别清空全部缓存。
- SSE事件只触发刷新，不作为正式状态来源。

## 5. 组件约束

- 页面文件原则上不超过300行；超过时必须拆分。
- 业务组件原则上不超过250行。
- Hook只处理一种清晰职责。
- 不在组件内定义大型静态配置、Schema或Mock数据。
- 不使用`any`逃避类型检查；确需未知输入时使用`unknown`并解析。
- 不使用索引作为可变列表Key。
- 不在渲染函数中发起副作用。
- 不把所有逻辑集中到一个全局Store。

## 6. 样式约束

- 设计令牌是颜色、间距、圆角、阴影和动效唯一来源。
- 禁止在业务组件中散落任意十六进制色值。
- Tailwind类名按布局、尺寸、文字、颜色、状态顺序组织。
- 复杂组件使用变体工具统一管理状态。
- 禁止大面积`!important`。
- 禁止通过全局CSS覆盖第三方组件内部结构。

## 7. 表单与编辑器

- Schema驱动校验。
- 字段错误贴近字段。
- 保存状态可见。
- 离开页面时处理未保存修改。
- 自动保存必须防抖且支持版本冲突。
- Markdown渲染禁用不可信HTML。
- 富文本输出先解析成受控节点，不执行模型返回脚本。

## 8. 日志和监控

前端错误事件至少包含：

- 构建版本。
- 路由。
- 当前角色。
- 项目、课时和节点业务ID。
- Trace ID。
- 错误码。
- 浏览器和设备类型。

不得上报提示词全文、教材正文、模型密钥或未脱敏个人数据。

## 9. Git和提交

- `main`受保护。
- 功能使用短分支和Pull Request。
- Commit使用`feat/fix/refactor/test/docs/chore`前缀。
- PR必须关联需求或变更单。
- PR必须包含截图或录屏、测试结果和影响范围。
- 禁止把`.env`、生产数据、密钥和大型生成资产提交到仓库。

## 10. 构建命令

```text
pnpm install --frozen-lockfile
pnpm dev
pnpm dev:mock
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm storybook
pnpm build
```

上述命令必须在README中可复制执行。
