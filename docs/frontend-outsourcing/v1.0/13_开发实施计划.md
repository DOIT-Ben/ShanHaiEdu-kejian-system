# 山海教育课件工作台 V2 前端实施计划

> 执行本计划前必须应用`15_已确认业务口径修订_2026-07-17.md`。本计划中的组件和测试已按动态结构、通用创作台及当前PPT/视频顺序修订。

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 交付一个可独立部署、支持Mock和真实API、覆盖教师端与管理端的生产级山海教育课件工作台前端。

**Architecture:** 使用React 19、TypeScript和Vite构建纯前端SPA，通过`/api/v2`访问FastAPI。TanStack Query保存服务端状态，Zustand只保存界面状态；OpenAPI生成DTO，SSE只通知刷新，MSW提供无后端开发模式。

**Tech Stack:** React 19、TypeScript、Vite、React Router、Tailwind、shadcn/ui、Radix UI、TanStack Query、Zustand、React Hook Form、Zod、Framer Motion、openapi-typescript、openapi-fetch、MSW、Vitest、Testing Library、Playwright、Storybook、pnpm。

## Global Constraints

- 使用pnpm并提交锁文件。
- TypeScript开启严格模式和`noImplicitAny`。
- 不使用Next.js和前端BFF。
- 不在浏览器保存会话Token、模型密钥和生产凭据。
- 不直接调用模型Provider。
- 页面不得直接调用`fetch`。
- 服务端项目和节点状态不得复制到Zustand。
- 生产构建不得启用MSW。
- 所有业务页面实现空、加载、错误、权限和响应式状态。
- 新增核心依赖必须经甲方书面批准。
