# 05 交互状态与文案规则

## 1. 标准节点状态

| 状态 | 教师文案 | 允许操作 |
|---|---|---|
| `locked` | 等待前置步骤 | 查看阻断原因 |
| `ready` | 可以开始 | 编辑输入、准备提示词 |
| `queued` | 排队中 | 查看任务、允许取消时取消 |
| `running` | 生成中 | 查看进度、离开页面 |
| `needs_review` | 待审核 | 查看、编辑、返修、批准 |
| `revision_required` | 需要返修 | 修改输入或提示词 |
| `approved` | 已批准 | 查看、创建新版本 |
| `stale` | 上游内容已变化 | 查看差异、重新生成或确认沿用 |
| `failed` | 生成失败 | 查看原因、按规则重试 |
| `blocked` | 当前步骤被阻断 | 查看解决方法 |
| `cancelled` | 已取消 | 重新开始 |
| `skipped` | 本项目已跳过 | 查看跳过原因 |

## 2. 必须实现的页面状态

- 首次空状态。
- 加载骨架屏。
- 自动保存中、已保存、保存失败。
- 排队、生成、Provider等待、下载和媒体处理。
- 待审核、已批准、需返修和已失效。
- 预算不足、Provider不可用、网络中断和权限不足。
- 文件上传失败、解析失败和部分候选成功。
- SSE断开、重连和轮询降级。

## 3. 统一生成交互

生成前弹出或展开费用确认区，显示：

- 业务模型名称。
- 高级模式下的Provider和真实模型。
- 候选数量。
- 关键参数。
- 预计费用范围。
- 是否允许备用模型。

已经提交付费任务后，不得以“重试”名义静默创建第二个付费任务。

## 4. 返修

### 快速返修

教师输入简短修改要求。系统基于原提示词创建新提示词版本，再创建新模型运行。

### 高级返修

教师编辑完整提示词和授权范围内参数。平台安全契约、输出Schema和资产白名单只读。

不得覆盖原PromptVersion、ModelRun和ArtifactVersion。

## 5. 自动保存

```text
本地修改
→ 800–1200ms防抖
→ 携带row_version保存
→ 服务端返回新row_version
```

遇到409冲突必须展示我的修改、服务端版本和差异，提供保留我的版本、使用服务端版本或创建副本。

## 6. 错误展示

- 字段错误在字段下方。
- 节点错误在当前画布。
- 任务错误在任务坞持续显示。
- 短期通知使用Toast。
- 阻断错误使用持久状态面板并显示解决动作。
- 页面渲染崩溃使用路由级Error Boundary。

不得把所有错误都显示成Toast。

## 7. 文案规则

- 面向教师使用教学和创作语言。
- 不显示`node_id`、`provider_task_id`、Schema路径等内部字段，除非进入管理员或调试详情。
- 错误文案采用“发生了什么＋是否扣费＋下一步怎么做”。
- 按钮使用明确动词：生成、保存草稿、批准并继续、重新校验、下载。
- 禁止“确定”“操作成功”“未知错误”等缺乏上下文的独立文案。

## 8. 可访问性

- 所有操作可通过键盘完成。
- 焦点样式清晰。
- 状态不能只靠颜色。
- 图标按钮具有可读名称和Tooltip。
- 视频播放器支持字幕开关和键盘控制。
- 动效遵守系统减少动态效果设置。
