# 通用创作中心

## 1. 创作首页

主标题“今天想创作什么”，提供画一张教学图片、做一段课堂视频、制作一套教学PPT三个大入口，以及继续上次创作、项目送来的待生成内容、最近作品和运行任务。

创作中心是一个全局入口，内部三种能力使用独立路由和模块。以后新增音频或数字人不增加全局导航项。

## 2. 两种界面密度

独立单项创作默认不显示左侧批次栏，中央空间全部用于作品。项目导入多个任务时才显示可收起任务列表。

创作台布局：

- 顶部：作品名称、来源、默认保存目标和主要操作；
- 左侧：仅批量模式显示任务项；
- 中央：当前作品和候选；
- 右侧：按需展开生成要求、参考素材和高级调整；
- 底部：极简任务状态。

## 3. 图片创作

生成前中央展示描述、参考图、比例、风格和数量，主按钮“开始生成”。模型参数和负面要求放入进一步调整。

生成后显示一个大尺寸当前候选和底部缩略图条。操作顺序为采用结果、提出修改、重新生成、保存到项目。不得把所有候选缩成同等小网格。

## 4. 视频创作

教师端使用“参考画面、画面怎样变化、镜头持续多久”。生成后中央为大播放器，候选位于下方。批量模式左侧显示镜头1、镜头2及其简单状态。

模型成功只表示候选完成。用户采用并保存到项目后，才显示“已保存为项目片段”。技术任务ID只在生成记录中出现。

## 5. PPT创作

左侧页面缩略图可收起，中央为大尺寸16:9页面，右侧只显示当前选中对象的调整。顶部保留返回、名称、保存、预览和导出。

封面未确认时，正文入口显示“请先确定封面风格”和一个去选择封面的按钮，不展示复杂流程图。

## 6. 生成要求

普通视图使用结构化生成要求：内容、风格、构图、角色、色彩、比例和禁止项。点击“查看和修改完整生成指令”后展示编译后的业务Prompt以及只读约束摘要。

## 7. 主操作状态

```text
生成前：开始生成
生成后：采用这个结果
采用后：保存到项目
保存后：已保存到具体项目位置
```

每个阶段只突出一个主操作。独立创作在保存时选择项目；项目导入创作默认带出来源项目和目标位置，但用户可以改为其他有权限项目或只下载。
