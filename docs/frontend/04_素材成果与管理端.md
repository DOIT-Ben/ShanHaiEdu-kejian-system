# 素材成果与管理端

## 1. 素材与成果

教师端名称为“素材与成果”，不是资产管理。

顶部三张当前成果卡展示教案、PPT和课堂导入视频。下方按全部、教学图片、PPT页面、视频镜头、音频字幕和文档筛选，并支持按课时、类型和使用位置查看。

三类九套导入设计作为教案附录产物展示在教案详情和文档筛选中，显示独立审核状态与当前选中方案；它不是第四个最终交付大卡。

中央使用视觉网格。点击素材后大预览，右侧按需展示名称、用途、来源、当前采用版本和主要操作。历史版本只在点击后展开。

批量选择底部操作为用于PPT、用于视频、去创作中心和下载。内部AssetSlot和ArtifactRelation不显示。

替换当前版本时明确说明影响位置，原版本保留；受影响PPTX或视频提示重新导出。未保存创作候选不进入该页面。

项目交付单独展示教案DOCX/PDF、PPTX、完整视频、字幕、质量报告和交付包，不展示中间素材。

## 2. 管理端导航

```text
返回教师端｜内容中心｜工作流｜模型服务｜运行与费用｜用户权限｜审计记录
```

管理端可以使用中等密度表格，但详情仍采用清晰分区。

## 3. 内容中心

内容中心管理教案结构、课时规则、生成指令、三类九套独立创意规则、最小课程锚定规则、PPT工作手册、视频规则、质量检查、画面风格和模型预设。

标准导入流程：上传、系统检查、预览变化、试运行、发布。动态结构必须提供真实教师端预览。发布版本不可原地覆盖，已有项目不自动迁移。

详情标签为内容、结构预览、测试案例、版本记录和使用情况。区分平台锁定、业务默认、用户可编辑和输出格式要求。

## 4. 工作流管理

第一阶段使用清晰步骤编排，不做无限画布。选中步骤配置能力包、上下文白名单、输出结构、人工门禁、自动化、预算、重试、跳过和过期范围。发布前执行依赖、循环、缺失包和测试项目检查。

## 5. 模型服务

管理端提供服务总览、供应商、模型目录、使用规则、连接测试和调用记录。业务节点引用逻辑能力，管理员配置主模型、备用模型、并发、超时和费用。

密钥只显示配置状态和尾号。测试请求不进入教师项目。

## 6. 运行、费用和审计

运行页突出任务积压、失败率、异常服务、费用异常和可处理动作，不做无意义大屏。审计记录内容包发布、工作流切换、模型规则修改、Prompt人工修改、批准驳回、跨项目保存、资产替换和下载。
