# 组件化与代码架构

## 1. 分层

```text
src/
├─ app/          启动、路由、Provider和权限
├─ pages/        路由页面，只做组合
├─ layouts/      全局、项目、创作台和管理端骨架
├─ features/     垂直业务模块
├─ entities/     前端领域模型和映射
├─ shared/       通用UI、API和工具
└─ generated/    OpenAPI生成代码
```

依赖只能从app/pages流向layouts/features，再到entities和shared/generated。Feature外部只能从公开入口引用，不得读取其他Feature内部实现。

## 2. 布局骨架

`GlobalAppShell`、`ProjectOverviewLayout`、`ProjectWorkbenchLayout`、`CreativeStudioLayout`和`AdminLayout`独立存在。布局使用插槽组合，不判断具体业务类型。

## 3. Feature

按home、projects、materials、lesson-division、lesson-plan、content-definition、prompt-editing、creation-batches、image-studio、video-studio、ppt-studio、project-results、generation-tasks、approvals、version-history、save-to-project和admin拆分。

每个Feature自包含api、components、hooks、mappers、model和tests。

## 4. 注册表扩展

- `nodeRendererRegistry`注册工作流作品渲染器。
- `contentFieldRegistry`注册rich_text、repeatable、math、timeline等动态字段。
- `artifactPreviewRegistry`注册图片、视频、PPT页和文档预览。
- `studioRegistry`注册图片、视频、PPT及未来创作能力。

禁止巨型`if/else`或`switch`按节点类型渲染。

## 5. 状态边界

| 状态 | 工具 |
|---|---|
| 服务端项目、作品、任务和版本 | TanStack Query |
| 表单草稿 | React Hook Form |
| 面板、缩放、选中等界面状态 | Zustand或局部状态 |
| 可分享页面状态 | URL |
| 长任务通知 | SSE触发Query刷新 |

不得把服务端数据复制到Zustand，不得由SSE直接覆盖完整对象，不得用`useEffect`手工同步两套状态。

## 6. API边界

OpenAPI生成客户端，经过entities或Feature mapper转换为View Model后进入组件。组件不得拼URL、直接调用Provider或依赖Provider原始响应。

## 7. 工程门禁

- TypeScript严格模式，禁止无理由`any`。
- 页面原则上不超过250行，业务组件不超过300行，超过400行必须拆分并说明。
- 颜色、间距、圆角和动效全部使用Design Token。
- 禁止复制页面、固定十二字段、临时特殊节点和业务数据写入全局Store。
- CI检查循环依赖、非法跨层引用、OpenAPI生成差异、测试、Storybook、构建和关键视觉回归。

新增能力应新增模块、注册项、合同和测试，不在原有页面继续堆条件。
