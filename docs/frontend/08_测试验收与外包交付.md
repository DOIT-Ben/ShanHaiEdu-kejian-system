# 测试、验收与外包交付

## 1. 外包开工要求

承接方先提交现有代码审计：路由、页面、组件、状态管理、重复代码、巨型组件、可保留部分、必须重构部分和删除清单。禁止在旧结构上继续追加补丁。

迁移顺序：Design Token和基础组件、全局框架、主页、项目工作台、创作中心、素材与成果、管理端、API任务、测试和旧代码清理。

## 2. 必须交付

- Figma源文件和组件库；
- React/TypeScript生产源码；
- Storybook；
- OpenAPI生成客户端；
- MSW场景；
- 单元、组件、视觉和端到端测试；
- 环境配置说明；
- 构建部署说明；
- 组件和路由清单；
- 删除旧实现清单；
- 全新环境交接记录。

## 3. 关键验收

- 主页有品牌表现、真实作品感和明确创作入口。
- 当前作品占功能页视觉中心，一个区域只有一个主操作。
- 教师端不暴露不必要专业术语。
- 项目工作台和通用创作中心使用不同布局骨架。
- 单项创作不显示无用批次栏；右侧高级设置默认收起。
- 两套不同教案Schema无需修改页面代码即可渲染。
- 项目创作包可导入创作中心，未保存候选不进入项目。
- PPT关键内容可编辑，视频候选保存后才显示已采用片段。
- 新作品类型通过注册表扩展，没有巨型节点条件组件。
- 服务端状态不复制到Zustand。
- 1440、1280、1024关键页面视觉回归通过。
- SSE断线重连、409冲突、部分成功、失败重试和刷新恢复通过。

## 4. 测试门禁

CI运行lint、typecheck、Vitest、Storybook build、生产build、Playwright关键流程、依赖边界、循环依赖和包体积检查。生产构建不得包含MSW、密钥或调试后门。

## 5. 关键端到端流程

- 新用户从主页上传教材并创建项目；
- 课时划分、动态教案生成、编辑和批准；
- 教案＋PPT、教案＋视频和全部分支；
- 项目待生成内容导入图片/视频/PPT创作；
- 项目批次保存提示词版本、生成、采用并写回CreationPackage固定槽位；
- 独立批次不携带项目上下文，采用后可下载，或显式选择有权限项目与槽位写回；
- 独立创作只下载；
- 封面门禁、正文纯白和PPTX导出；
- 视频shot局部失败、重试、保存clip和最终合成；
- 自动化暂停、人工修改和继续；
- 最终交付下载。
