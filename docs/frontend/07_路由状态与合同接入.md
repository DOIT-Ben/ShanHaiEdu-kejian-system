# 路由、状态与合同接入

## 1. 当前生产路由

```text
/login
/app
/app/projects
/app/projects/new
/app/projects/:projectId/setup
/app/projects/:projectId
/app/projects/:projectId/materials/:materialId?
/app/projects/:projectId/lessons
/app/projects/:projectId/assets
/app/projects/:projectId/artifacts/:artifactId
/app/projects/:projectId/jobs/:jobId
/app/projects/:projectId/lessons/:lessonId/work/:stepKey
```

以上清单与`apps/web/src/app/RuntimeApp.tsx`一致，是当前可用于Issue、PR和真实API验收的路由事实。`/app/login`和`.../workbench`不是生产路由。通配降级页不是业务能力，不能用于声明功能完成。

## 2. 规划路由


项目、课时、当前步骤、创作类型和批次必须反映在URL中。刷新和前进后退恢复当前位置。

## 3. 节点状态

前端合同使用`disabled`、`not_ready`、`ready`、`draft`、`queued`、`running`、`review_required`、`approved`、`partially_completed`、`failed`、`paused`、`cancel_requested`、`cancelled`、`stale`和`skipped`。

节点、生成任务、候选审核、保存操作和批准状态分别建模，不能用一个状态字段控制全部界面。

## 4. API和并发

- 基础前缀`/api/v2`。
- 异步任务返回`202`和`job_id`。
- 更新可编辑资源使用`ETag/If-Match`或`lock_version`。
- 创建、生成、批准、重试和保存操作使用`Idempotency-Key`。
- 前端使用游标分页和短期文件下载授权。

## 5. SSE

订阅全局、项目或任务事件流，支持`Last-Event-ID`、心跳和重连。事件只触发目标Query刷新，不携带完整业务对象。断线不停止后端任务。

## 6. 动态结构

页面接收`content_definition_version_id`、JSON Schema、UI Schema、data、字段权限和校验结果。通用渲染器支持text、rich_text、number、list、table、group、repeatable、reference、asset、math、rubric和timeline。未知字段类型显示明确不兼容错误，不得静默丢弃。

## 7. 文件

前端申请预签名地址直传对象存储，再调用完成接口。只有后端验证进入`ready`后才显示上传完成。下载每次重新申请短期授权。

共享合同以仓库根`contracts/`为准。
