# 产品范围与固定决策

## 1. 前端范围

包含教师主页、项目、教材与课时、动态教案、PPT、课堂导入视频、通用创作中心、素材与成果、任务、交付，以及内容中心、工作流、模型服务、权限和审计管理端。

不包含后端业务实现、数据库、Provider适配、媒体处理、密钥保存和正式费用结算。

## 2. 固定业务口径

- 一个项目对应一个小知识点教材，内部包含多个课时。
- 教案必选；PPT和视频是互相独立的可选分支。
- 教案正文和三类九套课堂导入设计是独立审核的兄弟产物；导出时方案集可作为教案附录。
- 教案由Content Definition Schema驱动，十二部分只是默认结构。
- 一键生成和按步骤辅助创作引用同一个WorkflowDefinition、项目、节点运行、任务、版本和资产，只由AutomationPolicy决定暂停位置。
- 独立创意生成不读取课程；锚定节点只补充最小课程回接；视频只读取教师选中方案快照，不读取教案正文。
- 项目以不可变CreationPackage把来源节点、上下文和固定目标槽位导入创作台；前端不得用可变共享对象绕过创作包。
- 项目批次和独立批次使用判别合同；项目批次不能改换目标，独立批次附加到项目时必须显式选目标并重新鉴权。
- 保存提示词版本、生成候选、采用候选和写回项目是四个独立动作；只有写回成功后候选才进入项目素材与成果。
- 视频顺序为母版剧本、粗分镜、视觉母图、图片资产、细分镜、候选片段、正式片段、声音和成片。
- 视频候选保存成功后才显示正式`clip_id`对应的教师语言“已采用片段”。
- PPT封面单独设计并先审核；正文固定纯白且每页有图片主视觉，最终PPTX关键文字和数学图形可编辑。
- Prompt在教师端称为“完整生成指令”，业务层可修改。
- 页面不显示内部状态机、Provider任务ID、资产位和关系表术语。

## 3. 体验硬原则

- 当前作品默认占功能页可用空间70%以上。
- 一个视觉区域只保留一个主按钮。
- 用户三秒内看懂正在做什么、看哪里、点哪里和改哪里。
- 直接在作品上修改；高级设置默认收起。
- 左侧用教师行动语言，流程只负责定位。
- 长任务离开页面继续运行。
- 没有真实进度时展示阶段，不伪造百分比。
- 未保存候选不污染项目。

## 4. 教师语言

| 内部概念 | 教师端显示 |
|---|---|
| CreationPackage | 待生成内容 |
| CreationBatch | 本次创作 |
| Artifact | 作品 |
| Prompt | 完整生成指令 |
| StyleContract | 画面风格 |
| NodeRun | 当前步骤 |
| review_required | 等待你确认 |
| stale | 内容已变化，建议更新 |
| shot_id | 镜头1、镜头2 |
| clip_id | 已采用片段 |
| Provider / provider_model | 不直接展示；教师只看到质量/速度档位 |

内部标识只在管理员或故障详情中显示。
