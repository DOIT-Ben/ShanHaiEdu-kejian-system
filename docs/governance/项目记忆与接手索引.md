# 项目记忆与接手索引

本文件是 ShanHaiEdu 的稳定接手路由，不是第二套产品规格、任务状态或历史摘要。它帮助新成员用最少阅读定位当前权威源；被链接文件的现行内容始终高于本文件中的概括。

## 职责和权威

- 维护者：仓库维护者和承担相关模块变更的任务负责人。
- 读者：首次接手、跨会话恢复、审查或交接 ShanHaiEdu 的开发者和智能体。
- 唯一路径：本文件只在 `docs/governance/项目记忆与接手索引.md` 原位维护。
- 唯一职责：保存稳定原则、事实路由、验证入口和反复出现的接手陷阱。
- 替换规则：若 [AGENTS](../../AGENTS.md) 与 [最短阅读入口](../START_HERE.md) 将本文件全部职责纳入机器可检验入口，应在同一 Pull Request 删除本文件及所有链接；不得保留副本。

不同事实按职责选择权威源，不把它们排成可以互相覆盖的线性层级：

| 事实类型 | 权威源 |
| --- | --- |
| 当前任务指令与操作约束 | 用户当前指令、[仓库规则](../../AGENTS.md)和被分配的 GitHub Issue |
| 产品预期与业务语义 | 已批准的产品/工作流文档及对应 Decision Issue |
| 跨模块机器边界 | OpenAPI、JSON Schema、事件合同和消费者测试 |
| 已实现运行行为 | 当前代码、Alembic 迁移、测试和实际运行结果 |
| 里程碑、阻塞与任务交接 | [CURRENT_STATUS](../../CURRENT_STATUS.md)、GitHub Issue 和 Pull Request |
| 历史原因与外部提示 | 定向查询的 Git 历史、外部长期记忆和会话冷证据，仅用于定位 |

产品预期、机器合同和实现是不同维度的事实，任何一方都不能静默覆盖另外两方。三者不一致时共同构成阻塞缺陷，按仓库规则停止扩展并更新 Issue，不能由本索引代替决策。

## 接手读取顺序

1. 读取 [README](../../README.md) 和 [AGENTS](../../AGENTS.md)，确认项目身份与强制规则。
2. 读取本索引，只建立权威源和模块边界地图。
3. 读取 [CURRENT_STATUS](../../CURRENT_STATUS.md)，获取里程碑、阻塞和下一出口。
4. 实时拉取远端，核验工作区、上游、Issue、Pull Request 和任务分支。
5. 按 [最短阅读入口](../START_HERE.md) 只进入本任务相关的文档、合同、代码和测试。
6. 只有当前资料无法解释回归、责任或既有决定时，才定向查询 Git 历史或外部记忆。

外部记忆只负责把接手者带回这个仓库和本索引。它不能证明某个分支、提交、Pull Request、端口、Provider 或运行结果仍然有效。

## 验证入口

按风险选择最小相关命令，再扩大到任务要求的完整门禁：

```powershell
uv run pytest tests\unit
uv run pytest tests\integration
uv run pytest tests\contract
uv run ruff format --check .
uv run ruff check .
uv run pyright
pnpm contracts:check
uv run python scripts\check_repository.py
uv run python scripts\check_tracked_secrets.py
git diff --check
```

- 数据库任务必须运行 Alembic 和真实 PostgreSQL 集成检查，不能用 SQLite 代替。
- Provider 适配与阶段出口必须运行明确授权的真实冒烟；普通测试不得访问真实 Provider。
- 合并后从 `origin/main` 复验，不能把任务分支或本地环境结果冒充主线证据。
- 只有实际执行成功的命令才能写入交接；历史评论和外部记忆中的通过记录只用于定位。

## 记忆边界

仓库内允许保存：

- 稳定的产品原则及其 canonical 链接；
- 模块边界、验证入口和冲突处理规则；
- 已经反复证明、仍会影响未来任务的接手陷阱。

仓库内禁止保存：

- 某次任务的分支、提交、Pull Request、端口、进程和测试计数；
- 完整聊天、会话摘要、日记、临时计划和模型原始响应；
- 本机绝对路径、个人资料、密钥、Provider 凭据和真实用户素材；
- 复制的产品字段、OpenAPI 或 Schema 正文；
- 与 [CURRENT_STATUS](../../CURRENT_STATUS.md)、Issue 或 Pull Request 竞争的实时状态。

外部长期记忆只能保存仓库身份、本索引的仓库相对路径和“使用前实时复核”规则。原生会话记忆属于冷证据，不自动晋升为当前项目事实。

## 维护责任

- 产品原则变化时，先更新批准的产品/工作流/合同来源，再判断本索引中的路由概括是否需要同步。
- 新增、移动或删除 canonical 入口时，在同一 Pull Request 更新本索引、README、AGENTS 和 START_HERE 中受影响的链接。
- 里程碑、任务、分支和运行状态变化时，不更新本文件；更新 CURRENT_STATUS、Issue 或 Pull Request。
- 每次修改运行 [仓库治理脚本](../../scripts/check_repository.py)，确认必需章节、链接和记忆边界通过。
- 本文件超过 300 行、开始承载业务字段或出现第二个项目记忆文件时，必须拆回原有 canonical 来源并删除重复内容。
