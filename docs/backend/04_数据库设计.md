# 数据库设计

本文件只描述当前 Alembic 唯一迁移头、SQLAlchemy 模型和数据库集成测试已经实现的数据库事实。未来表、候选字段和迁移方案进入 GitHub Issue，不在本文件中预先声明为现行设计。

维护责任：修改 ORM 模型或 Alembic 迁移的后端任务必须同步本文件。被替代时就地修改；不得保留版本副本。

## 1. 权威边界

数据库事实按以下顺序核验：

1. `alembic/versions/` 与数据库当前迁移头。
2. `apps/api/*/models.py`。
3. PostgreSQL 集成测试和约束测试。
4. 本文件的人类可读说明。

当前迁移注册 50 张业务表；`alembic_version` 是迁移工具表，不计入业务模型。生产环境禁止 ORM 自动建表，所有结构变化必须使用 Alembic。

## 2. 通用约束

- PostgreSQL 16 是业务状态唯一真相；Redis 只承载队列和短期协调。
- 业务主键使用 UUID；应用新建业务记录使用 UUIDv7。
- 租户数据必须带 `organization_id` 或通过外键可确定所属组织。
- 可变聚合使用 `created_at`、`updated_at`、`created_by`、`updated_by`、`lock_version` 和软删除字段；追加写事实不强行套用可变审计列。
- JSONB 只保存动态合同、快照、规则和脱敏元数据，不替代稳定外键、唯一约束和索引。
- 发布版本、输入快照、用量记录和其他追加写事实通过约束或触发器禁止原地重写。
- 对象存储位置、哈希和短期签名分离；预签名 URL 不入库。

## 3. 身份、项目与执行策略

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `organizations` | 租户 | `slug` 唯一；状态受控 |
| `users` | 跨组织用户身份 | 邮箱唯一；状态受控 |
| `organization_members` | 用户与组织关系 | 活动成员关系唯一 |
| `principals` | 用户或系统审计主体 | 组织内主体唯一且可停用 |
| `project_members` | 项目角色 | `(project_id,user_id)` 唯一；角色为 owner/editor/reviewer/viewer |
| `projects` | 项目固定内容发布与工作流版本 | 同组织活动 `project_no` 唯一；软删除；乐观锁 |
| `project_automation_policies` | 版本化执行控制 | `(project_id,policy_version)` 唯一；mode 仅为 guided/automatic |

`projects.automation_mode` 是旧合同留下的物理兼容列。当前运行时只从最新 `project_automation_policies.mode` 投影 `execution_mode`，不得读取该旧列作业务判断。兼容列的最终退役由 [Issue #91](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/91) 处理。

项目创建同时写入项目、owner 成员关系和首条自动化策略。工作流运行固定策略快照，不随项目后续策略修改而变化。

## 4. 内容发布与工作流

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `content_packages` | 内容包稳定身份 | `package_key` 唯一 |
| `content_package_versions` | 内容包不可变版本 | 包内语义版本唯一；校验和唯一；发布后不可变 |
| `content_package_item_versions` | 包内已校验内容项载荷 | 包版本内 `item_key` 唯一；包发布后不可变 |
| `content_releases` | 一组内容包版本的发布 | `release_key` 唯一；发布后不可变 |
| `content_release_items` | Release 到包版本的挂载 | 同 Release 的 `mount_key` 唯一 |
| `runtime_default_versions` | 新项目采用的Release与工作流默认版本 | 按 `runtime_key` 追加版本；只允许指向已发布事实 |
| `content_definition_versions` | 结构化内容定义 | 包版本内 `definition_key` 唯一；校验和固定 |
| `workflow_definitions` | 工作流稳定身份 | `workflow_key` 唯一 |
| `workflow_definition_versions` | 已发布工作流图 | 定义内版本号唯一；校验和固定 |
| `workflow_runs` | 项目工作流实例 | `(project_id,run_no)` 唯一；固定 Release、工作流版本和策略快照 |
| `branch_runs` | 课时分支运行 | `(workflow_run_id,lesson_unit_id,branch_key)` 唯一 |
| `node_runs` | 节点执行实例 | 运行号、状态、触发类型、活动产物和 stale 事实 |
| `node_input_snapshots` | 节点输入事实 | `(node_run_id,input_key)` 唯一；追加写 |

仓库黄金内容包通过正式发布服务写入Manifest、111个内容项、22个内容定义投影和47节点工作流目录，并创建不可变Release。项目创建从最新`runtime_default_versions`固定Release与工作流版本；发布新默认不会更新既有项目。Alembic只种入兼容旧运行时的首条默认版本，黄金包发布由显式管理命令执行。

该迁移在尚未发布新内容时可正常降级；一旦存在包内容项或追加的运行时默认版本，降级会明确拒绝，避免删除不可变发布事实。此后回退只能通过恢复应用代码或发布新的前向Release完成，不能删除发布历史。

## 5. 课时、教材与文件资产

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `lesson_units` | 项目课时 | 项目内 lesson_key、position 分别唯一；状态受控 |
| `lesson_branch_configs` | 教案、三类九套、PPT、视频分支开关 | `(lesson_unit_id,branch_key)` 唯一；四个固定分支键 |
| `source_materials` | 项目教材与来源材料 | 绑定项目、上传状态和当前文件资产 |
| `upload_sessions` | 临时上传会话 | 项目、材料、对象键、预期 MIME/大小/哈希和过期时间 |
| `material_parse_versions` | 教材解析版本 | 材料内版本号唯一；任务绑定唯一；成功/失败终态不可重写 |
| `file_assets` | 组织内文件资产聚合 | 同组织 `asset_key` 唯一；当前版本指针 |
| `file_asset_versions` | 不可变对象版本 | 资产内版本号唯一；对象位置唯一；哈希和核心元数据不可覆盖 |
| `project_asset_slots` | 项目作品目标槽位 | `(project_id,slot_key)` 唯一；one/many 基数与目标合同 |
| `asset_bindings` | 槽位到文件版本的追加写绑定 | 活动位置唯一；解绑只做单向停用 |

当前 `file_assets` 只有组织边界，没有完整的个人/项目/共享 ACL。独立创作引用和跨项目共享必须等待 [Issue #82](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/82) 决策，不能仅凭同组织关系放行。

## 6. Artifact、审核与 Prompt 快照

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `artifacts` | 项目作品聚合 | 活动 `(project_id,artifact_key)` 唯一；草稿/提交/批准指针 |
| `artifact_drafts` | 可编辑草稿分支 | 活动 `(artifact_id,draft_branch)` 唯一；乐观锁 |
| `artifact_versions` | 不可变作品版本 | `(artifact_id,version_no)` 唯一；内容哈希与来源快照 |
| `approvals` | 追加写审核事实 | submit/approve/request_changes/accept_stale/revoke |
| `artifact_relations` | 版本依赖图 | 关系身份唯一；禁止自环和跨项目关系 |
| `context_snapshots` | 节点上下文冻结 | 每个 NodeRun 一条；内容哈希；禁止更新和删除 |
| `prompt_snapshots` | Prompt 编译与公共预览冻结 | 每个 NodeRun 一条；完整 Prompt 与脱敏 preview 分离 |

Artifact 批准后清除当前 submitted 指针；request_changes 只允许当前 in_review 版本。依赖关系类型当前由数据库和服务共同限制，统一机器枚举由 [Issue #86](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/86) 完成。

## 7. 通用创作、任务与模型审计

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `creation_packages` | 项目节点进入创作台的不可变来源包 | 固定项目、工作流、节点、上下文、Prompt 和目标规则 |
| `creation_package_items` | 创作包条目 | 包内 item_key、position 分别唯一 |
| `creation_batches` | 项目或独立创作批次 | source_kind 为 project/standalone；独立批次必须有 owner_user_id |
| `creation_items` | 批次中的创作条目 | 固定批次、当前 Prompt 版本和采用结果 |
| `creation_prompt_versions` | 教师业务 Prompt 版本 | 条目内版本号唯一；引用资产列表和输出合同固定 |
| `generation_jobs` | Worker 可恢复任务 | 项目任务或独立创作任务；状态、租约、取消和幂等事实 |
| `generation_results` | 生成候选 | 任务内候选号唯一；可绑定文件资产版本 |
| `adoptions` | 候选采用事实 | 条目与结果固定；采用不等于项目写回 |
| `save_to_project_operations` | 创作结果原子写回 | 幂等键、目标项目/槽位、授权快照和绑定结果 |
| `generation_attempts` | 模型调用尝试 | 节点尝试号和组织 request_id 唯一；终态字段一致 |
| `usage_records` | 一次 Attempt 的用量与费用事实 | 每个 Attempt 一条；追加写；费用和延迟非负 |
| `node_execution_recovery_facts` | 已校验节点输出的隔离恢复事实 | 租户/节点/Attempt唯一；固定owner、request、Schema与快照哈希；24小时过期；T2成功后删除 |

普通 CI 只使用确定性 Fake。真实 Provider 响应、密钥和完整 Prompt 不进入这些审计表。历史 running Attempt 的恢复租约和并发序号由 [Issue #90](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/90) 处理。

通用节点执行器由 [Issue #89](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/89) 实现。Provider调用不进入数据库事务；校验后的pending文本先把Attempt终态、UsageRecord和隔离恢复事实原子checkpoint，随后T2原子写Artifact、关系、NodeRun和合同允许的CreationPackage并删除恢复事实。完整Prompt、原始模型正文和Provider响应仍不进入审计表或公共合同。

## 8. 幂等、Outbox 与事件流

| 表 | 当前职责 | 关键约束 |
| --- | --- | --- |
| `idempotency_records` | 命令响应重放 | `(organization_id,scope,idempotency_key)` 唯一；请求哈希冲突返回 409 |
| `outbox_events` | PostgreSQL 内的待投递事件 | 租约、重试、可用时间和脱敏错误 |
| `event_stream_entries` | 项目 SSE 可重放事件 | `(project_id,sequence_no)` 唯一；Last-Event-ID 按项目恢复 |

所有幂等重放必须先执行当前授权；撤销项目成员关系后不得返回历史成功响应。Outbox 的长期职责与独立创作事件序列仍需 [Issue #83](https://github.com/DOIT-Ben/ShanHaiEdu-kejian-system/issues/83) 决策。

## 9. 核心事务边界

以下操作必须在单个 PostgreSQL 事务中完成，外部对象存储或 Provider 调用必须通过可恢复状态衔接：

1. 创建项目、owner 成员和初始 AutomationPolicy。
2. 上传确认、不可变 FileAssetVersion、解析 Job 和事件写入。
3. 保存 Artifact 草稿或提交版本及事件。
4. 审核 Artifact、更新当前指针、传播 stale 和写事件。
5. 保存 Creation Prompt 版本并更新条目指针。
6. 采用候选与更新 CreationItem。
7. SaveToProjectOperation、AssetBinding、条目状态和事件写回。
8. Attempt 终态与 UsageRecord 审计。
9. 节点执行T2中的Artifact、ArtifactRelation、NodeRun、可选CreationPackage与恢复事实删除。

## 10. 当前不存在的数据库能力

以下名称或概念不属于当前迁移，不能在代码审查、接口说明或进度报告中宣称已实现：

- 独立的 capability/projection 定义表、内容包文件表和内容评测运行表。
- 独立的 PPT 页、视频剧本、镜头、音轨、字幕、TTS 或交付包领域表。
- model_profiles、provider_configs、model_routes 等可配置 Provider 管理表。
- intro_selections、audit_logs、domain_events 等独立表。
- 项目预算字段、automation_policy_id 当前指针或 owner_user_id 项目列。

需要新增这些能力时，先建立 Decision 或实施 Issue，再同步迁移、模型、合同、测试和本文件。禁止只在文档中预建未来数据库。

## 11. 迁移与验证

数据库变更至少验证：

```powershell
uv run alembic upgrade head
uv run pytest tests\integration\test_migrations.py
uv run pytest tests\integration
uv run pyright
uv run ruff check .
```

破坏性迁移、无法确定归属的数据回填和生产数据删除必须单独获得明确授权。开发和测试清理不得使用 `docker compose down -v`。
