# 数据库设计

## 1. 总体约束

PostgreSQL 16 是业务状态唯一真相。应用使用 SQLAlchemy 2 和 Alembic；生产环境禁止 ORM 自动建表。表名和列名使用 `snake_case`，主键使用 UUIDv7。

所有可变业务表默认包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `uuid` | UUIDv7 主键 |
| `organization_id` | `uuid` | 租户边界；平台表可为空 |
| `created_at` / `updated_at` | `timestamptz` | UTC 时间 |
| `created_by` / `updated_by` | `uuid` | 系统任务使用系统用户 |
| `lock_version` | `integer` | 乐观锁，从 1 开始 |
| `deleted_at` | `timestamptz null` | 软删除；历史表通常不允许删除 |

JSONB 只承载动态内容、供应商元数据和规则快照，不能替代应建立外键、唯一约束或索引的核心关系。

## 2. 身份与租户

| 表 | 关键字段 | 关键约束 |
| --- | --- | --- |
| `users` | `email`, `display_name`, `status`, `last_login_at` | 规范化邮箱唯一；状态为 active/disabled |
| `organizations` | `name`, `slug`, `status`, `settings_json` | `slug` 全局唯一 |
| `principals` | `organization_id`, `principal_type`, `display_name`, `status` | 阶段0只建立可审计的系统主体；用户主体随认证任务扩展 |
| `organization_members` | `organization_id`, `user_id`, `role`, `status` | 活动成员 `(organization_id,user_id)` 唯一 |
| `project_members` | `project_id`, `user_id`, `role` | `(project_id,user_id)` 唯一；项目角色不得高于组织授权 |

项目角色：`owner`、`editor`、`reviewer`、`viewer`。平台管理权限独立存储，不借用项目角色。

## 3. 项目、课时与教材

### `projects`

`id`, `organization_id`, `project_no`, `title`, `subject`, `school_stage`, `grade`, `textbook_edition`, `knowledge_point`, `default_language`, `status`, `content_release_id`, `workflow_definition_version_id`, `automation_policy_id`, `budget_amount`, `budget_currency`, `owner_user_id`, 通用审计字段。

约束：同组织 `project_no` 唯一；首期 `subject='primary_math'`；归档项目禁止创建新运行，恢复后才可编辑。

### `lesson_units`

`organization_id`, `project_id`, `lesson_key`, `position`, `title`, `scope_summary`, `objective_summary`, `estimated_minutes`, `source_division_version_id`, `status`, `lock_version`。

约束：`(project_id,lesson_key)` 和 `(project_id,position)` 分别唯一；`position > 0`；课时物理删除受历史版本引用限制。

### `lesson_branch_configs`

`lesson_unit_id`, `branch_key`, `enabled`, `settings_json`, `lock_version`。

约束：`(lesson_unit_id,branch_key)` 唯一；`branch_key` 为 lesson_plan/intro_options/ppt/video；lesson_plan 默认启用且不可关闭，intro_options 默认启用但可明确关闭，PPT 和视频默认关闭并可独立开关。`enabled=false` 映射工作流 `disabled`；`skip_reason` 只属于一次明确跳过的运行记录，不进入分支配置。

### `source_materials`

`project_id`, `material_kind`, `file_asset_id`, `original_filename`, `mime_type`, `upload_status`, `confirmed_at`, `confirmed_by`。

`upload_sessions` 保存 `project_id`, `material_id`, `storage_bucket`, `storage_key`, 预期 MIME/大小/SHA-256、状态和过期时间；短期预签名 URL 不入库。

### `material_parse_versions`

`source_material_id`, `version_no`, `status`, `parser_name`, `parser_version`, `content_json`, `page_count`, `text_checksum`, `validation_report_json`, `file_asset_version_id`。

约束：`(source_material_id,version_no)` 唯一；成功解析版本不可更新内容；页段引用在 `content_json` 中必须符合注册 Schema。

解析状态为 `pending → running → succeeded | failed`。失败重试创建新版本，不复用或重置失败记录；`succeeded` 和 `failed` 终态由数据库阻止更新。

## 4. 内容运行时

### `content_packages`

`package_key`, `name`, `package_type`, `owner_scope`, `status`。`package_key` 稳定且全局唯一。

### `content_package_versions`

`content_package_id`, `semantic_version`, `runtime_constraint`, `manifest_json`, `archive_asset_version_id`, `checksum`, `status`, `validated_at`, `published_at`。

约束：`(content_package_id,semantic_version)` 唯一；发布后不可变；`checksum` 唯一防止重复包伪装成新版。

### `content_package_files`

`content_package_version_id`, `path`, `media_type`, `sha256`, `content_text`, `asset_version_id`。

约束：包内 `path` 唯一；只能二选一保存文本或资产引用；禁止绝对路径和 `..`。

### `content_releases`

`release_key`, `name`, `status`, `published_at`, `published_by`, `notes`。发布后不可变。

首期迁移写入固定UUID的内置Release、内容定义和工作流版本，并显式回填既有项目。应用注册表只提供新项目默认指针；默认值变化不得更新既有项目的固定外键。

### `content_release_items`

`content_release_id`, `content_package_version_id`, `mount_key`, `priority`。

约束：同一发布 `mount_key` 唯一；依赖图必须无环。

### `content_definition_versions`

`definition_key`, `content_package_version_id`, `schema_json`, `ui_schema_json`, `export_mapping_json`, `validation_rules_json`, `checksum`。

`definition_key + content_package_version_id` 唯一。已被项目引用的定义不允许物理删除。

### `capability_definition_versions`

`capability_key`, `content_package_version_id`, `display_name`, `responsibility`, `input_contract_refs_json`, `output_contract_refs_json`, `handbook_refs_json`, `workflow_entry_key`, `required_capabilities_json`, `human_gate_rules_json`, `checksum`。

约束：同包 capability_key 唯一；输入输出合同必须存在；发布后不可变；业务能力不得包含供应商密钥或可执行代码。

### `projection_definition_versions`

`projection_key`, `content_package_version_id`, `source_definition_key`, `target_media_type`, `template_text`, `registered_renderer_id`, `rules_json`, `checksum`。

约束：模板或已注册 renderer 二选一；内容包不能声明任意模块路径；目标媒体类型必须在平台允许列表。

### `content_evaluation_runs`

`content_package_version_id`, `suite_key`, `status`, `case_count`, `passed_count`, `report_json`, `started_at`, `finished_at`, `runner_version`。发布所需评测未通过时数据库状态机禁止发布。

## 5. 工作流与自动化

### `workflow_definitions` / `workflow_definition_versions`

定义表：`workflow_key`, `name`, `domain`, `status`。版本表：`workflow_definition_id`, `version_no`, `graph_json`, `input_contract_json`, `status`, `checksum`, `published_at`。

约束：`(workflow_definition_id,version_no)` 唯一；发布前校验 DAG、节点键唯一、合同引用存在；发布后不可变。

### `automation_policies`

`policy_key`, `name`, `scope`, `version_no`, `mode`, `rules_json`, `status`, `published_at`。

约束：`(policy_key,version_no)` 唯一；运行实例保存策略快照而非仅保存当前 ID。

### `workflow_runs`

`project_id`, `workflow_definition_version_id`, `content_release_id`, `automation_policy_snapshot_json`, `run_no`, `status`, `started_at`, `completed_at`, `current_event_seq`。

约束：`(project_id,run_no)` 唯一；每项目最多一个 `status in ('active','paused')` 的主运行，使用部分唯一索引。

启动运行时重新校验项目固定的已发布工作流图、节点键、依赖和合同引用；应用服务先检查活动运行，数据库部分唯一索引再处理并发竞争。

### `branch_runs`

`workflow_run_id`, `lesson_unit_id`, `branch_key`, `status`, `started_at`, `completed_at`。

约束：`(workflow_run_id,lesson_unit_id,branch_key)` 唯一。

### `node_runs`

`workflow_run_id`, `branch_run_id`, `node_key`, `run_no`, `status`, `trigger_type`, `automation_policy_snapshot_json`, `active_artifact_version_id`, `stale_reason_json`, `started_at`, `finished_at`, `last_error_code`。

约束：`(branch_run_id,node_key,run_no)` 唯一；同一节点最多一个 queued/running/cancel_requested 运行；状态转换由应用服务校验。

### `node_input_snapshots`

`node_run_id`, `input_key`, `source_type`, `source_id`, `source_version_id`, `content_hash`, `snapshot_json`。

约束：`(node_run_id,input_key)` 唯一；执行后不可修改。

首期快照采用只追加语义，数据库禁止更新和删除；需要修正输入时创建新的NodeRun和新快照。

## 6. 产物、版本、审核与资产

### `artifacts`

`organization_id`, `project_id`, `lesson_unit_id`, `branch_key`, `artifact_key`, `artifact_type`, `content_definition_version_id`, `current_draft_id`, `current_submitted_version_id`, `current_approved_version_id`, `status`, `stale_reason_json`。

约束：活动记录 `(project_id,artifact_key)` 唯一；三个当前引用必须指向本 Artifact。

### `artifact_drafts`

`organization_id`, `artifact_id`, `draft_branch`, `content_json`, `validation_report_json`, `based_on_version_id`, `autosaved_at`, `lock_version`。

约束：活动 `(artifact_id,draft_branch)` 唯一；更新必须带旧 `lock_version`。

### `artifact_versions`

`organization_id`, `artifact_id`, `version_no`, `content_json`, `content_hash`, `render_summary_json`, `source_kind`, `source_node_run_id`, `context_snapshot_id`, `prompt_snapshot_id`, `validation_report_json`, `created_at`, `created_by`。

约束：`(artifact_id,version_no)` 唯一；`content_hash` 非空；版本行只可插入不可更新内容。

### `context_snapshots`

`scope_type`, `scope_id`, `bindings_json`, `content_hash`, `created_at`。`bindings_json` 保存来源版本 ID 和必要内容副本或不可变定位。

### `prompt_snapshots`

`scope_type`, `scope_id`, `template_refs_json`, `layers_json`, `editable_prompt`, `user_diff_json`, `compiled_prompt`, `request_schema_json`, `content_hash`, `created_at`。

访问完整提示词需单独权限；审计日志仅记录快照 ID 和哈希。

### `approvals`

`organization_id`, `artifact_version_id`, `node_run_id`, `action`, `actor_type`, `actor_user_id`, `comment`, `quality_evidence_json`, `policy_snapshot_json`, `created_at`, `created_by`。`created_by` 引用可审计 Principal，使系统审核与用户审核使用同一追加写事实。

约束：追加写；批准动作通过事务更新 Artifact 当前批准指针。

### `intro_selections`

`lesson_unit_id`, `intro_option_set_version_id`, `option_key`, `choice_mode`, `recommendation_score`, `selected_by`, `selected_at`, `is_active`, `supersedes_selection_id`。

约束：每课时只有一个活动选择；`choice_mode` 为 teacher_selected/policy_default；引用的版本必须是 `intro_option_set` 且 option_key 存在；选择记录追加写，切换时停用旧记录。该表不保存或复制完整教案。

### `artifact_relations`

`organization_id`, `from_artifact_version_id`, `to_artifact_version_id`, `relation_type`, `binding_key`, `impact_scope_json`, `created_at`, `created_by`。

约束：关系二元组、关系类型和 `binding_key` 唯一；禁止自环和跨项目关系；应用层在租户级事务锁内检查依赖图，防止并发写入形成派生环。

### `file_assets`

`organization_id`, `asset_key`, `asset_kind`, `current_version_id`, `status`, `retention_class`。同组织 `asset_key` 唯一。

### `file_asset_versions`

`file_asset_id`, `version_no`, `storage_bucket`, `storage_key`, `mime_type`, `byte_size`, `sha256`, `etag`, `width`, `height`, `duration_ms`, `page_count`, `scan_status`, `metadata_json`, `derived_from_version_id`, `created_at`。

约束：`(file_asset_id,version_no)` 唯一；`(storage_bucket,storage_key)` 唯一；`byte_size >= 0`；对象位置、哈希和核心元数据不可原地替换。宽高、时长和页数允许从空值补写一次，写入后不可覆盖。

### `project_asset_slots`

`project_id`, `lesson_unit_id`, `slot_key`, `asset_type`, `cardinality`, `required`, `status`, `target_contract_json`。

约束：`(project_id,slot_key)` 唯一；`cardinality` 为 one/many。

### `asset_bindings`

`project_asset_slot_id`, `file_asset_version_id`, `source_generation_result_id`, `source_artifact_version_id`, `save_operation_id`, `position`, `is_active`, `bound_at`, `unbound_at`。

约束：单值槽位活动绑定部分唯一；多值槽位 `(slot_id,position)` 活动唯一；解绑历史不删除。

## 7. 通用创作与生成

### `creation_packages`

`package_key`, `source_project_id`, `source_lesson_unit_id`, `source_node_run_id`, `package_type`, `status`, `style_contract_version_id`, `context_snapshot_id`, `prompt_snapshot_id`, `target_rules_json`, `source_stale_at`, `created_at`。

约束：`(creation_package_id,item_key)` 和 `(creation_package_id,position)` 唯一。

### `creation_batches`

`organization_id`, `owner_user_id`, `creation_package_id`, `studio_type`, `title`, `status`, `settings_json`, `project_hint_id`。`project_hint_id` 只用于便捷选择，不构成资产归属。

### `creation_items`

`creation_batch_id`, `item_key`, `position`, `title`, `current_revision_id`, `status`, `selected_result_id`。

### `creation_item_revisions`

约束：`(creation_item_id,revision_no)` 唯一；旧修订不可变。

### `generation_jobs`

`organization_id`, `project_id`, `source_material_id`, `creation_item_revision_id`, `node_run_id`, `job_type`, `status`, `progress_percent`, `progress_message`, `requested_candidates`, `completed_candidates`, `idempotency_key`, `request_hash`, `priority`, `lease_owner`, `lease_expires_at`, `cancel_requested_at`, `started_at`, `finished_at`, `last_error_code`。

约束：`(organization_id,idempotency_key)` 唯一；候选计数非负；至少关联创作修订或节点运行之一。

### `generation_attempts`

`generation_job_id`, `attempt_no`, `model_route_id`, `provider_config_id`, `provider_task_id`, `status`, `request_snapshot_json`, `response_snapshot_ref`, `submitted_at`, `last_polled_at`, `finished_at`, `error_code`, `error_details_json`, `lease_owner`。

约束：`(generation_job_id,attempt_no)` 唯一；有值时 `(provider_config_id,provider_task_id)` 唯一。

### `generation_results`

`generation_job_id`, `generation_attempt_id`, `candidate_no`, `status`, `file_asset_version_id`, `structured_output_json`, `quality_score`, `quality_evidence_json`, `safety_status`, `provider_metadata_json`, `selected_at`, `saved_at`, `expires_at`。

约束：`(generation_job_id,candidate_no)` 唯一；媒体结果必须有文件版本，文本结果必须有结构化内容；质量分范围 0–100。

### `save_to_project_operations`

`organization_id`, `idempotency_key`, `source_generation_result_id`, `target_project_id`, `target_slot_key`, `replace_mode`, `status`, `created_binding_id`, `error_code`, `completed_at`。

约束：`(organization_id,idempotency_key)` 唯一；一项成功操作只能生成一个主绑定；结果、项目和槽位租户必须一致。

## 11. 可靠性与审计

### `idempotency_records`

`organization_id`, `scope`, `idempotency_key`, `request_hash`, `response_status`, `response_body_json`, `resource_type`, `resource_id`, `expires_at`。唯一 `(organization_id,scope,idempotency_key)`；同键不同请求摘要返回冲突。

### `domain_events`

`aggregate_type`, `aggregate_id`, `aggregate_version`, `event_type`, `payload_json`, `occurred_at`。用于业务追溯，不等于完整事件溯源。

### `outbox_events`

`event_id`, `topic`, `aggregate_type`, `aggregate_id`, `payload_json`, `status`, `available_at`, `published_at`, `attempt_count`, `last_error`。`event_id` 唯一；领取使用 `FOR UPDATE SKIP LOCKED`。

### `audit_logs`

`organization_id`, `actor_type`, `actor_user_id`, `action`, `resource_type`, `resource_id`, `request_id`, `ip_hash`, `before_summary_json`, `after_summary_json`, `created_at`。追加写并单独设置保留期；不得保存密钥和完整敏感正文。

### `event_stream_entries`

`organization_id`, `project_id`, `sequence_no`, `event_type`, `resource_type`, `resource_id`, `summary_json`, `created_at`。项目内 `sequence_no` 单调递增，用于 SSE 断线续传。

## 12. 核心事务

### 批准产物并传播过期

同一事务中锁定 Artifact，追加 Approval，更新当前批准版本，读取真实依赖关系，标记下游 Artifact/NodeRun 过期，追加领域事件与 Outbox。若任一步失败全部回滚。

### 保存创作结果到项目

同一事务中验证幂等、租户、权限、结果状态和目标合同；创建或复用文件资产身份，创建项目槽位绑定，记录来源关系，更新保存操作和节点满足度，追加 Outbox。对象存储复制如有需要，先使用可恢复的 `pending` 文件版本并由异步任务确认，不跨网络保持数据库事务。

### 发布内容包

锁定待发布版本，验证所有引用和评测通过，创建不可变 Release/Item，切换“新项目默认发布”指针，写审计与 Outbox。已有项目指针不变化。

## 13. 索引与查询

- 所有租户业务表索引以 `organization_id` 开头。
- 项目工作台：`node_runs(branch_run_id,node_key,run_no desc)`、`artifacts(project_id,lesson_unit_id,branch_key)`。
- 任务面板：`generation_jobs(organization_id,status,created_at desc)`；租约扫描 `status,lease_expires_at`。
- 创作结果：`generation_results(generation_job_id,status,candidate_no)`。
- 项目资产：`project_asset_slots(project_id,lesson_unit_id,asset_type)` 和活动绑定部分索引。
- SSE：`event_stream_entries(project_id,sequence_no)` 唯一。
- JSONB 仅为真实查询路径建立表达式或 GIN 索引，禁止无差别索引所有 JSON。
- 软删除唯一约束使用 `WHERE deleted_at IS NULL` 部分索引。

## 14. 数据库级防线

- 使用 Check Constraint 限定状态枚举、金额、计数、时长和质量分。
- 对跨表语义不变量使用事务应用服务加延迟约束或触发器审计，不能只靠前端。
- 数据访问层强制租户过滤；可在成熟后增加 PostgreSQL RLS 作为纵深防御，但不能在未测试情况下仓促启用。
- 生产迁移必须可前滚、可回滚或提供补偿脚本；大表变更采用扩展—回填—切换—清理。
- 每日备份并验证恢复；关键环境启用时间点恢复。对象存储与数据库备份策略必须匹配，避免只恢复其中一边。
