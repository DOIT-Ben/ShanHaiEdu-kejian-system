## 固定技术基线

- API：Python 3.12、FastAPI、Pydantic v2、SQLAlchemy 2、Alembic
- 数据库：PostgreSQL 16
- 队列与短期状态：Redis + Dramatiq；业务真相不得只存在 Redis
- 对象存储：兼容 S3 的私有桶，通过短期签名地址访问
- 课件处理：LibreOffice/OnlyOffice、Playwright、FFmpeg
- 运行形态：模块化单体 API + 独立 Worker，先不拆微服务
- 通信：同步 REST + 异步任务 + Outbox + SSE
- 可观测性：OpenTelemetry、结构化日志、指标、错误追踪

## 后端必须保证的业务不变量

1. 一个项目只对应一个小学数学知识点，但可拆为多个课时。
2. 教案是每课时必选主干；PPT、导入视频可独立开关。
3. 三类九套导入设计是教案正文之外的独立附录产物；独立创意与课程锚定必须分节点执行。
4. 视频只消费被选导入方案快照，不消费完整教案。
5. 项目保存所有中间产物、审核结果和最终交付物。
6. 通用创作中心可脱离项目使用；只有用户执行“保存到项目”后，结果才成为项目资产。
7. 提示词、上下文、模型路由和输出都必须留快照，确保可复现。
8. 已批准版本不可原地覆盖；修改必须产生新版本。
9. 上游新版本只有在批准后，才会精准标记实际依赖它的下游为过期。
10. 任何模型生成任务都可重试、可取消、可审计，不因页面刷新而丢失。
11. 自动化模式可替用户推进，但不得绕过结构校验、质量门槛和高风险确认。
12. 代码不硬编码教案字段、提示词、锚点、工作流或模型供应商。
