# 三类九套课堂导入设计

## 1. 定位

三类九套是每课时默认生成的课堂导入创意库，位于完整教案正文之后，作为独立、可替换、可单独审核的附录产物 `intro_option_set`。它不是教案十二部分中的固定第十三项，也不影响教案正文独立通过审核。

教师明确不需要课堂导入设计时可以关闭。教师已有明确创意时可以只完善该创意，不强制扩成九套。

两条路径使用同一个`intro_option_set`：默认路径记录`generation_mode=default_nine`并包含九套；已有创意路径记录`generation_mode=refine_existing`并只包含一套。它们不是两种产品模式，也不复制审核、选择或视频输入合同。

## 2. 三类锚点定义

| 类别 | 独立创意的核心 | 锚点机制 | 回到课堂的第一问 |
| --- | --- | --- | --- |
| 科普 `science` | 一个值得观察、解释或追问的现象、未知、尺度变化或反常结果 | 在现象尚未解释的时刻，用本课能研究的一个关系作为唯一回接 | “这为什么会发生/怎样解释？” |
| 应用 `application` | 一个真实任务、限制条件、选择、比较或完成目标的需要 | 在必须作出决定或完成任务的时刻，用本课能解决的一个问题作为唯一回接 | “怎样完成/选哪种方法更合适？” |
| 故事 `story` | 人物目标、变化、阻碍、误会、悬念或未完成行动 | 在关键转折或问题尚未解决的时刻，用本课能继续探索的一个问题作为唯一回接 | “接下来怎么办/问题出在哪里？” |

“故事”不等于必须有强冲突；“科普”不强行人物化；“应用”不等于把教材题换皮。每类生成三套实质不同方案，共九套。

## 3. 两阶段生成

### 阶段 A：独立创意

只允许读取锚点类别、创意规则、儿童安全规则和教师通用媒介偏好。禁止读取教材、知识点、课时目标、教案、PPT 和课程标题。

先生成九个独立创意核心：即使观众不知道学科和课题，也能理解基本事件；去掉课程回接后，仍有观看、思考或参与价值。

### 阶段 B：最小课程锚定

独立创意冻结后，另一个节点读取已批准课时范围、教材证据和不得提前讲授边界，只允许补充：`course_anchor`、`classroom_first_question`、`handoff_moment`、`must_not_preteach`、适配理由、风险和推荐度。

该节点不得改写独立概念、钩子或观看价值，不能让知识点反向生成创意。每套只能有一个最小课程锚点；视频必须在交接时刻停止，把正式发现、定义、方法和结论留给课堂。

## 4. 每套方案字段

| 字段 | 语义 |
| --- | --- |
| `option_key` | 稳定键，如 `INTRO-SCI-01` |
| `category` | science / application / story |
| `title` | 简洁、可辨识的方案名 |
| `independent_concept` | 脱离课程仍成立的完整创意 |
| `hook` | 开场发现、任务、变化、悬念或问题 |
| `viewer_value` | 去掉课程回接后仍值得观看/参与的原因 |
| `suggested_medium` | 视频、图片、实物、提问、表演或混合 |
| `duration_seconds` | 建议课堂导入时长 |
| `replacement_field_key` | 可替换教案教学过程中的哪个基础导入 |
| `course_anchor` | 全方案唯一最小课程回接 |
| `classroom_first_question` | 结束后教师提出的第一问 |
| `handoff_moment` | 独立内容停止、教师接回课堂的具体时刻 |
| `must_not_preteach` | 视频不得提前给出的定义、方法、结论和探究步骤 |
| `fit_reason` | 与本课、年级和课堂节奏的适配理由 |
| `risks` | 误导、超时、泄题、设备或组织风险 |
| `recommendation_score` | 1–100，九套最高分唯一 |
| `recommendation_reason` | 适配度、节奏、制作成本和风险的综合理由 |

## 5. 选择协议

1. 前端先展示按推荐度排序的九套卡片和打分依据。
2. 教师可以查看全文、修改允许字段、指定任意一套或明确采用默认首选；人工指定记录为`teacher_selected`。
3. 未指定且自动化策略允许时，采用唯一最高分方案；选择方式记录为`policy_default`。只有该选择方式要求方案等于唯一最高分。
4. 选择结果写入独立 `intro_selection` 运行记录，不回写教案正文和已批准方案集版本。
5. 教案正文和 PPT 不等待教师完成选择；视频分支必须等待选择。

## 6. 视频交接合同

视频只接收被选方案的不可变快照：

- 方案键、标题和类别；
- `independent_concept`、`hook`、`viewer_value`；
- `course_anchor`、`classroom_first_question`、`handoff_moment`；
- `must_not_preteach`；
- 推荐媒介、时长和必要风险。

视频不接收完整教案、教学目标、教学过程、板书、作业或 PPT。它可以忠实发展被选创意为一份完整母版剧本，但不能重新选择创意、改写锚点或越过交接时刻。

## 7. 硬性质量门

- 不知道本课内容的观众仍能理解独立事件。
- 去掉课程锚点后仍有独立价值。
- 九套不是只换标题、角色或场景。
- 科普、应用、故事各恰好三套；最高推荐分唯一。
- 每套只有一个课程锚点，并能定位具体交接时刻。
- 不提前讲出课堂需要学生发现的内容。
- 不因小学受众默认使用儿童、教师、教室或卡通角色。
- 方案中不包含分镜、旁白、字幕、资产清单、模型参数或费用策略。
