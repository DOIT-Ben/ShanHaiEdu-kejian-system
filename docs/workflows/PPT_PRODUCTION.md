# PPT 生产流程与页级合同

## 1. 目标

把批准教案和教材证据转换为可直接上课、可逐页审核、可局部返修、关键内容可编辑的 PPTX。最终效果是“白底上的图片化课堂课件”，不是讲义式信息图，也不是把整页 AI 图片塞进 PPT。

## 2. 固定顺序

```text
批准教案与教材证据
→ 内容分析和PPT大纲
→ 逐页教学任务与四层设计
→ 封面主视觉候选
→ 批准封面并锁定视觉合同
→ 正文图片资产任务包
→ 批量生图和逐项采用
→ 逐页混合装配与预览
→ 单页编辑、返修和审核
→ 可编辑PPTX与PDF渲染
→ 技术、教学和视觉验收
```

大纲、封面和最终 PPTX 是强制审核门。自动化可以准备候选和正文，但不得在封面未批准时锁定整套视觉风格。

## 3. 内容分析和大纲

分析只读取批准教案、教材证据、课件用途和教师偏好，输出：受众、目标、重点难点、课堂顺序、互动点、适合图片化的内容、必须保持可编辑的内容、建议页数和每页唯一教学任务。

大纲必须覆盖导入、概念建构、方法形成、例题或探究、应用练习、辨析和总结中实际需要的部分，不机械凑页。每页只承担一个主要教学任务，并保留教材知识顺序和例题逻辑。

## 4. 逐页四层合同

每个 `page_key` 保存一个结构化页设计：

### 第一层：画布 `canvas`

- 封面：允许主题统领型主视觉，不使用正文的白底组件式结构。
- 正文：背景固定纯白 `#FFFFFF`，不使用纹理、渐变、满屏场景图或无关背景装饰。
- 记录画幅、安全区、留白和投影屏可读性。

### 第二层：图片主视觉与元素 `visual`

每页必须有明确图片主视觉。先判断学生最需要看见的是数量关系、整体与部分、比较、转化、单位“1”、变化、操作还是生活应用，再选择：

- `textbook_reconstruction`：教材情境高清重构，不能照搬原始插图和版式；
- `original_asset`：原创实物、纸艺模型、立体教具、微型场景、生活物件或数据装置；
- `mixed`：教材认知连续性 + 原创图片资产 + 可编辑数学图形。

即使内容抽象，也优先寻找图片化表达，再用可编辑数学图形辅助。禁止只用文字、公式、表格、箭头、色块和矢量图形冒充主视觉。

### 第三层：可编辑教学信息 `content`

标题、正文、问题、数字、百分数、算式、公式、题面、答案、标注、线段图、百格图、比例条、进度条和关键数据默认使用可编辑文本或 PPT 图形。图片生成提示词禁止依赖准确文字、数字、公式、字幕和 Logo。

### 第四层：排版与互动 `layout`

记录标题、主视觉、说明、问题、操作区和反馈区的位置、层级、对齐、间距、进入顺序和教师备注。元素边界清楚，不深度穿插或相互咬合，方便移动、替换和抠图。

## 5. 页级必需字段

- `page_key`、`position`、`page_type`；
- `teaching_task`、`source_refs`、`student_focus`；
- `canvas`；
- `visual_decision`、`image_strategy`、`main_visual_description`；
- `asset_requirements`：每项稳定键、用途、提示词、负面约束和目标槽位；
- `editable_text_blocks`；
- `editable_math_shapes`；
- `layout_spec`、`interaction_spec`、`speaker_notes`；
- `validation_rules` 和当前校验结果。

所有正文页必须明确 `background_mode=solid_white` 和至少一个主视觉资产要求。封面必须 `page_type=cover` 且 `background_mode=cover_art`。

## 6. 封面与视觉合同

封面独立生成 2–4 个候选，表达课题核心、教学情境和公开课吸引力，避免夸张冲突、无关奇观和廉价模板。教师采用一张后创建 `ppt_style_contract`：

- 主图风格、材质、造型和光线；
- 强调色及使用比例；
- 标题/正文字体层级和字号范围；
- 插画与照片处理方式；
- 组件圆角、描边、阴影和间距；
- 正文白底规则、禁止形式和图文分治规则。

正文继承封面视觉语言，而不是复制封面背景。更换已批准封面会使视觉合同和受影响正文资产过期，不直接删除已完成页面。

## 7. 图片资产进入创作中心

页设计批准后，系统按资产要求生成不可变图片 `CreationPackage`。同一套 PPT 共用视觉合同和一致性键；每个资产仍是独立条目，教师可一键导入图片创作中心并发生成。

创作中心可以修改提示词、参考图和模型；被采用结果通过 `SaveToProjectOperation` 保存到对应页面资产槽位。未保存候选不进入 PPT 文档。单页可补图、换图和重做，不重跑全套。

## 8. 混合装配与编辑

装配器按四层合同创建真实 PPT 元素：

- 底层：纯白画布或封面主视觉；
- 中层：可替换图片资产和组件；
- 上层：可编辑文本、数字、数学图形和标注；
- 备注层：教师提示和来源信息。

整页图片可用于快速预览，但不能作为默认最终页。装配后前端展示与实际 PPTX 同源的页面预览，教师可直接修改文字、替换图片、调整位置或重新生成单页。

## 9. 验收

- 教学范围、顺序和例题逻辑与批准教案/教材一致；
- 每页一个教学任务且有明确主视觉；
- 封面有设计，正文全部纯白背景；
- 数学表达准确，关键文字和图示可编辑；
- 无文字溢出、过小、遮挡、错位、裁切或低清；
- 页间风格统一但版式不机械重复；
- 互动、练习和总结数量符合真实教学需要；
- PPTX 可打开、可编辑，PDF/缩略图与 PPTX 渲染一致；
- 单页变化只精准使该页和当前整套导出版本过期。
